# Zenodo Archiving Checklist

1. Create a public GitHub repository named `RPS27L_MASLD_bioinformatics`.
2. Push this folder as the repository content.
3. Confirm that raw matrices, processed objects, local paths and working notes
   are not tracked by Git.
4. Create a GitHub release, for example `v1.0.0`.
5. Link the GitHub repository to Zenodo and archive the release.
6. Add the Zenodo DOI to `CITATION.cff` and the README after DOI assignment.
7. If the manuscript cites a specific software release, cite the Zenodo DOI
   rather than a mutable branch URL.
